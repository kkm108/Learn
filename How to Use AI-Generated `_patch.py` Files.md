## 1. What Is a Patch File?

When you ask an AI assistant to fix a bug, add a feature, or refactor code, it sometimes cannot rewrite your entire project — especially if the project is large. Instead, it provides a **patch script**: a small Python file named something like:

```
fix_login_bug_patch.py
add_dark_mode_patch.py
refactor_api_handler_patch.py
update_config_patch.py
```

A patch file is a **self-contained Python script** that:

- Opens one or more of your existing source files
- Finds specific text (a function, a block, a line, a setting)
- Replaces it with new, corrected content
- Saves the result back to disk

Think of it like a surgical instrument: instead of rewriting a whole file, it makes **precise, targeted edits** at exactly the right location.

---

## 2. How Patch Files Work

Most AI-generated patch files follow one of these patterns:

### Pattern A — Read / Replace / Write (most common)

```python
with open("src/app.py", "r") as f:
    content = f.read()

old = '''def login(user, password):
    return check(user, password)'''

new = '''def login(user, password):
    if not user or not password:
        raise ValueError("Credentials cannot be empty")
    return check(user, password)'''

content = content.replace(old, new)

with open("src/app.py", "w") as f:
    f.write(content)

print("✅ Patch applied successfully.")
```

### Pattern B — Regex-based replacement

Used when the exact whitespace or minor details may vary:

```python
import re

with open("config.yaml", "r") as f:
    content = f.read()

content = re.sub(r'timeout:\s*\d+', 'timeout: 30', content)

with open("config.yaml", "w") as f:
    f.write(content)
```

### Pattern C — Multiple files in one patch

A single patch script may touch several files at once:

```python
files_to_patch = {
    "src/auth.py": (...),
    "src/utils.py": (...),
    "tests/test_auth.py": (...),
}
```

### Pattern D — Append / Prepend

```python
with open("requirements.txt", "a") as f:
    f.write("\nrequests==2.31.0\n")
```

## 4. Before You Apply Any Patch — Safety First

> ⚠️ **Always do these steps before running any patch script.**

### Step 1 — Read the patch file first

Open the `.py` file in any text editor (Notepad, VS Code, nano, etc.) and read it from top to bottom. Ask yourself:

- What files does it open?
- What text is it looking for (the `old` variable)?
- What is it replacing it with (the `new` variable)?
- Does anything look unexpected or unrelated to your request?

### Step 2 — Back up your project

#### Option A: Git (strongly recommended)

```bash
git add .
git commit -m "Before applying patch: fix_login_bug"
```

Now even if everything breaks, one command restores you:

```bash
git checkout .
```

#### Option B: Manual backup (if not using Git)

Copy your entire project folder:

```
my_project/          ← original (keep this safe)
my_project_backup/   ← your copy
```

On Linux/macOS:

```bash
cp -r my_project my_project_backup
```

On Windows (Command Prompt):

```cmd
xcopy /E /I my_project my_project_backup
```

### Step 3 — Confirm the working directory

Patch files use **relative paths** (e.g., `"src/app.py"`). This means the terminal must be in the **root of your project** before you run the patch, or the script will not find your files.

```bash
# Check where you are
pwd          # macOS / Linux
cd           # Windows

# Move to your project root
cd /path/to/your/project
```

## 5. Step-by-Step: Applying a Patch

Follow these steps every time.

### Step 1 — Save the patch file

Save the patch script the AI gave you into your **project root** directory (next to your `src/`, `app.py`, `package.json`, etc.).

```
my_project/
├── src/
│   └── app.py
├── requirements.txt
└── fix_login_bug_patch.py   ← save it here
```

### Step 2 — Open a terminal in your project root

- **VS Code:** Terminal → New Terminal
- **Windows:** Right-click the folder → "Open in Terminal" (or use `cd`)
- **macOS:** Right-click → "New Terminal at Folder"
- **Linux:** Right-click → "Open Terminal Here"

### Step 3 — Run the patch

```bash
python fix_login_bug_patch.py
```

If `python` isn't found, try:

```bash
python3 fix_login_bug_patch.py
```

On Windows you may also use:

```cmd
py fix_login_bug_patch.py
```

### Step 4 — Read the output

A well-written patch prints a success or failure message:

```
✅ Patch applied successfully to src/app.py
```

or

```
❌ ERROR: Pattern not found in src/app.py — patch was NOT applied.
```

If you see an error, see the [Troubleshooting](https://claude.ai/chat/e1a776b0-7c63-4f6d-aeb5-0d7dcdd39709#11-troubleshooting) section.

### Step 5 — Inspect the changed files

Open the files that were patched and verify the changes look correct.

### Step 6 — Test your project

Run your test suite or manually test the feature that was patched.

```bash
pytest                  # Python projects
npm test                # Node.js projects
./gradlew test          # Java/Kotlin projects
```

## 7. Anatomy of a Patch File

Here is a fully annotated example of what a typical AI patch looks like:

```python
#!/usr/bin/env python3
"""
Patch: fix_login_bug
Target: src/auth/login.py
Description: Adds input validation before the authentication check.
Generated by: Claude (Anthropic)
"""

import os   # ← May import standard-library modules. No pip install needed.
import sys

# ── 1. Define the target file ─────────────────────────────────────────────────
TARGET = "src/auth/login.py"   # Path relative to your project root.

# ── 2. Safety check — abort if the file doesn't exist ────────────────────────
if not os.path.exists(TARGET):
    print(f"❌ ERROR: '{TARGET}' not found.")
    print("   Make sure you are running this script from the project root.")
    sys.exit(1)

# ── 3. Read the current file contents ────────────────────────────────────────
with open(TARGET, "r", encoding="utf-8") as f:
    content = f.read()

# ── 4. Define what to find (must match the file EXACTLY, including indentation)
OLD = '''def login(user, password):
    return authenticate(user, password)'''

# ── 5. Define the replacement ─────────────────────────────────────────────────
NEW = '''def login(user, password):
    if not user or not password:
        raise ValueError("Username and password are required.")
    return authenticate(user, password)'''

# ── 6. Perform the replacement ────────────────────────────────────────────────
if OLD not in content:
    print("❌ ERROR: Expected code block not found in the file.")
    print("   The file may have already been patched, or its contents differ")
    print("   from what the AI saw. No changes were made.")
    sys.exit(1)

patched = content.replace(OLD, NEW, 1)   # The '1' replaces only the first match.

# ── 7. Write the result back ──────────────────────────────────────────────────
with open(TARGET, "w", encoding="utf-8") as f:
    f.write(patched)

print(f"✅ Patch applied successfully to '{TARGET}'.")
```

Key things to notice:

|Part|What it does|
|---|---|
|`TARGET`|The file(s) that will be modified|
|`OLD`|The exact text to search for — must match perfectly|
|`NEW`|What replaces `OLD`|
|`content.replace(OLD, NEW, 1)`|The actual substitution|
|The `if OLD not in content` guard|Prevents silent failures|
|`encoding="utf-8"`|Ensures special characters are handled correctly|

## 11. Troubleshooting

### ❌ `python: command not found`

**Cause:** Python is not installed, or not in your system PATH.

**Fix:**

```bash
python3 fix_patch.py     # try python3 instead
py fix_patch.py          # try py on Windows
```

If none work, reinstall Python and check "Add to PATH" during setup.

---

### ❌ `FileNotFoundError: [Errno 2] No such file or directory: 'src/app.py'`

**Cause:** The terminal is not in the project root, so the relative path in the patch cannot be resolved.

**Fix:**

```bash
pwd                        # see where you are
ls                         # list files here
cd /path/to/your/project   # move to the right directory
python fix_patch.py        # try again
```

---

### ❌ `Pattern not found` or `Expected code block not found`

**Cause:** The exact text the patch is looking for does not exist in your file. This usually means:

1. Your file was already modified after the AI read it.
2. The file uses different whitespace, tabs vs. spaces, or line endings.
3. The patch was already applied once and the code has changed.

**Fix — Option A:** Ask the AI to regenerate the patch against your current file. Paste your current file content and say: _"The patch failed because the code has changed. Here is my current file — please update the patch."_

**Fix — Option B:** Apply the change manually. The patch file clearly shows you what `OLD` looks like and what `NEW` should be. Find `OLD` in your editor, select it, and replace it with `NEW` by hand.

---

### ❌ `UnicodeDecodeError`

**Cause:** The file contains special characters and the patch script doesn't specify an encoding.

**Fix:** Open the patch file in your editor and find the `open()` calls. Add `encoding="utf-8"` (or `encoding="latin-1"` for legacy files):

```python
# Change this:
with open("app.py", "r") as f:

# To this:
with open("app.py", "r", encoding="utf-8") as f:
```

---

### ❌ `IndentationError` or `SyntaxError` after patching

**Cause:** The replacement code has inconsistent indentation. Python files are whitespace-sensitive. Mixing tabs and spaces causes this.

**Fix:**

1. Open the patched file.
2. Find the changed section (use `git diff`).
3. Manually fix the indentation so it matches the surrounding code.
4. Or ask the AI: _"The patch caused an IndentationError at line X — please fix it."_

---

### ❌ The patch ran but nothing changed

**Cause:** The `replace()` silently found zero matches and wrote the file unchanged. This happens when the `if OLD not in content` guard is missing from the patch.

**Fix:** Check that the `OLD` text exists in your file. Open the file in your editor, copy the exact section you want replaced, and compare it character-by-character to what is in the `OLD` variable of the patch. Even a single extra space or missing newline will cause a mismatch.

---

### ❌ `PermissionError: [Errno 13] Permission denied`

**Cause:** The file is read-only, or on Windows it may be open in another program.

**Fix:**

```bash
# Linux / macOS — make the file writable
chmod u+w src/app.py
python fix_patch.py

# Windows — close any other program that has the file open, then retry
```

---

### ❌ Line endings (CRLF vs LF) cause the patch to fail on Windows

**Cause:** The AI wrote the patch with Unix line endings (`\n`) but your file uses Windows line endings (`\r\n`), causing the `OLD` string not to match.

**Fix — Option A:** Open the patch in VS Code, click "CRLF" in the bottom-right status bar, switch to "LF", save, and retry.

**Fix — Option B:** Add this to the patch's `open()` calls:

```python
with open("app.py", "r", newline="") as f:
```

## 12. FAQ

**Q: Is it safe to run a patch file from an AI?**

A: Generally yes — AI-generated patch files only use the Python standard library (`os`, `sys`, `re`, `pathlib`) and do nothing beyond reading and writing text files in your project. Always read the script before running it. Never run a patch that installs packages, makes network requests, or accesses files outside your project directory without understanding why.

---

**Q: Do I need to install anything with pip before running a patch?**

A: Almost never. Standard patches use only modules built into Python. If a patch imports a third-party library (e.g., `import autopep8`), it will tell you in a comment, or the AI will mention it explicitly.

---

**Q: Can I run a Python patch even if my project is JavaScript / Go / Rust?**

A: Yes. The patch is written in Python, but it edits plain text files. It does not care what language those files are written in. A Python script can patch a `.ts`, `.go`, `.rs`, `.yaml`, or `.html` file just fine.

---

**Q: What if I don't have Python at all?**

A: You can apply the patch manually:

1. Open the patch file and look at `OLD` and `NEW`.
2. Open the target file in your code editor.
3. Find the text from `OLD` (use Ctrl+F / Cmd+F).
4. Replace it with the text from `NEW`.
5. Save.

---

**Q: Can I apply the same patch twice?**

A: No. After the first application, the `OLD` text no longer exists in the file — it was replaced by `NEW`. Running the patch a second time will print a "pattern not found" error and make no changes. That is safe, but unnecessary.

---

**Q: The patch modified a file I didn't expect it to. What do I do?**

A: Use `git diff` to see exactly what changed. If the change is unwanted, restore the file with `git checkout <filename>` or copy from your backup. Then ask the AI to regenerate a more targeted patch.

---

**Q: The AI gave me the changes as inline code, not a patch file. Which should I prefer?**

A: Either works. Patch files are better for large or precise changes because they automate the edit exactly. Inline code is fine for small snippets you can copy-paste yourself. Ask for a patch file when the change is complex or touches multiple locations.

---

**Q: Can a patch file create brand-new files?**

A: Yes. Some patch scripts create entirely new files rather than modifying existing ones:

```python
with open("src/new_module.py", "w", encoding="utf-8") as f:
    f.write("""# new_module.py\n...\n""")
print("✅ Created src/new_module.py")
```

This is also common for adding migration files, test files, or configuration templates.

---

**Q: How do I request a patch file specifically from an AI?**

A: Add one of these to your prompt:

- _"Please give me a `_patch.py` file I can run to apply this change."_
- _"Don't rewrite the whole file — give me a patch script instead."_
- _"I have a large codebase; please provide a targeted patch file."_

---

**Q: What if the patch is very long or complex?**

A: Break it down. Ask the AI to split the patch into smaller, logical steps — one patch per concern. Apply and test them one at a time.

## 14. Glossary

|Term|Meaning|
|---|---|
|**Patch**|A targeted modification to an existing file|
|**Patch file / `*_patch.py`**|A Python script that applies a patch automatically|
|**Project root**|The top-level directory of your project, where you run commands from|
|**Relative path**|A file path like `src/app.py` that is relative to wherever you run the script|
|**`OLD`**|The exact text the patch looks for in your file|
|**`NEW`**|The replacement text that takes the place of `OLD`|
|**Diff**|A side-by-side comparison of what changed (`git diff`)|
|**Git checkout**|A command to restore a file to its last committed state|
|**Encoding**|How text characters are stored in bytes; `utf-8` is the universal standard|
|**CRLF / LF**|Windows vs. Unix line endings; mismatches can cause patches to fail|
|**Idempotent**|A patch is idempotent if running it twice produces the same result as once (good patches are designed this way)|
